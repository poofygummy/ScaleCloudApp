## Summary of changes in iOSClient

### Modified files
- NCLogin.swift
- NCLogin.storyboard
- `iOSClient/Account/Account Settings/NCAccountSettingsModel.swift`
- `iOSClient/Account/Account Settings/NCAccountSettingsView.swift`
- NCAutoUpload.swift
- SceneDelegate.swift
- NCPreferences.swift

### What changed

1. **Tailscale/ScaleCloud login flow**
   - Added custom login UI fields in `NCLogin.storyboard` for ScaleCloud/Tailscale accounts:
     - `scUsernameInput`
     - `scPasswordInput`
     - `scLoginButton`
   - Added corresponding outlets and styling in `NCLogin.swift`.
   - Added logic so when the entered server URL matches the special `toth-adattar` host pattern:
     - the normal address/login UI is hidden
     - the Tailscale username/password UI is shown
     - submit via the new arrow button or Return key triggers `performTailscaleLoginFromUI()`
   - Added host detection helpers:
     - `isToCsaCloud(_:)`
     - `isTailscaleAddress(_:)`

2. **Account settings for ScaleCloud**
   - Extended `NCAccountSettingsModel` with `isScaleCloudAccount` based on the same `toth-adattar` host logic.
   - Added a new settings section in `NCAccountSettingsView`:
     - “Watched Download Folders”
     - Links to a new ScaleCloud folder management view
     - Only shown for ScaleCloud accounts

3. **Auto-upload integration**
   - Updated `NCAutoUpload.swift` to:
     - detect ScaleCloud accounts
     - scan additional watched download folders for new files
     - enqueue downloads via a new helper path
     - preserve source collection context so Camera vs Screenshots can map to custom remote paths
   - Added a ScaleCloud-specific remote path mapping for uploads:
     - screenshots -> `/Képernyőmentések`
     - camera/other -> `/Saját Fényképek és Videók/`

4. **App launch auto-sync setup**
   - `SceneDelegate.swift` now:
     - checks photo library authorization
     - runs `setupScaleCloudAutoSync()` on launch
     - auto-configures Tailscale accounts for auto-upload
     - enforces settings like image/video auto-upload enabled and WWAN upload disabled
     - sets camera upload folder/filename defaults for ScaleCloud accounts
     - runs opportunistic check for new files in watched download folders on autoupload

5. **Preference storage**
   - `NCPreferences.swift` now supports:
     - storing and retrieving `ScaleCloudWatchedDownloadBookmarks`
     - generic string-array preference helpers
   - This supports persistence of security-scoped bookmark data for watched folders.

### New untracked files
These were created but are not yet committed:
- `iOSClient/Account/Account Settings/ScaleCloudWatchedFoldersModel.swift`
- `iOSClient/Account/Account Settings/ScaleCloudWatchedFoldersView.swift`
- ScaleCloudDownloadsHelper.swift

### Overall intent
The changes are clearly targeted at adding a ScaleCloud/Tailscale-specific path through:
- login UI customization
- account detection
- auto-upload setup
- watched downloads handling

scalecloud-ios/bridge.go (the Go source that becomes the XCFramework via gomobile bind)

• Complete implementation of the two exported functions that Swift calls: StartProxy(hostname, stateDir string) (int, error) and StopProxy() error.
• OAuth client ID + secret are hardcoded in init() and injected via TS_OAUTH_CLIENT_ID / TS_OAUTH_CLIENT_SECRET environment variables.
• isTailscaleAddress(host) — exactly the same logic Android uses (.ts.net suffix OR 100.64.0.0/10 CGNAT range).
• ensureTSNodeActive(hostname, stateDir) protected by nodeMX mutex — lazily creates the single *tsnet.Server (Ephemeral: false, ControlURL set to the official control plane, Logf = Discard).
• The real engine is inside StartProxy:
  • Listens on 127.0.0.1:0 (random free port).
  • Creates a goproxy.ProxyHttpServer.
  • Replaces its Transport.DialContext with custom logic: if the target host is a Tailscale address → call ensureTSNodeActive + tsNode.Dial(...); otherwise use a normal net.Dialer.
  • The proxy server is stored in the package-level proxyServer variable.
• StopProxy takes both proxyMX and nodeMX locks, shuts down the tsnet node (if any) and the http server (graceful shutdown with 5s timeout), and clears all state.
• Lots of package-level vars + two mutexes (proxyMX, nodeMX) for the same "only one proxy / one tsnet node ever" guarantee that Android has with its static fields.

scalecloud-ios/NextcloudKit/Sources/NextcloudKit/NKSession.swift (the only file modified in the NextcloudKit fork)

• New static state (direct port of Android's OwnCloudClient statics):
  • proxyClients: [WeakURLSession]
  • cleanupTimer: Timer?
  • proxyPort: Int
  • lock: NSLock
  • Inner WeakURLSession helper class (so the array doesn't keep sessions alive forever).
• applyProxySettings() — the central method:
  • Creates the exact state directory <app support>/tailscale (matching Android's <filesDir>/tailscale).
  • Calls Scalecloud.startProxy("ios-scalecloud-client", stateDir) (the gomobile-generated binding).
  • Returns the connectionProxyDictionary (HTTP + HTTPS both pointed at 127.0.0.1 + the port).
• checkAndCleanup() + registerSession(_:) — the lifecycle system:
  • Every time an NKSession creates any URLSession, it registers it.
  • A repeating timer (every 60 s) prunes dead sessions.
  • When the last one disappears → calls Scalecloud.stopProxy() and resets the port.
• proxyDictionary is now a stored let on every NKSession instance (populated in init before any sessions are built).
• In the init of NKSession:
  • self.proxyDictionary = NKSession.applyProxySettings() is called first (exact analog to the Android OwnCloudClient constructor calling applyProxySettings immediately).
• Every single one of the many URLSessionConfiguration objects (sessionData, sessionDataNoCache, all the background ones, the lazy *Ext ones, download, upload, WWAN, etc.) gets:

config.connectionProxyDictionary = proxyDictionary   // or self.proxyDictionary
  Unconditionally — every session goes through the proxy, exactly like Android.
• registerSession(...) is called on every one of those sessions.
• All the new code has very explicit comments saying "ScaleCloud: analogous to ... in OwnCloudClient.java" and "mirrors Android".

Other small but related changes in the scalecloud-ios tree that affect the NextcloudKit/ScaleCloud integration

• project.pbxproj was switched from an XCRemoteSwiftPackageReference (the official NextcloudKit 7.3.2 branch) to an XCLocalSwiftPackageReference pointing at the local NextcloudKit/ folder.
• .gitignore update to ignore local build artifacts of the custom NextcloudKit package.
• A short note added in the top-level README about the local package.
